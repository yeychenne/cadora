Cadora

The neutral conductor

One conductor for every coding agent — drive the top frontier models through a single workflow, on the subscriptions you already pay for, following the AI-DLC method, with proof you can ship.

open source · MITv0.9.0a capstone, not a platform

Cadora sits above the coding-agent CLIs and drives them through a workflow you declare — so the how of delivery stays yours. It is the conductor, not the agent: it doesn't implement the agent loop, and it isn't an in-IDE assistant. It's the layer that turns “an AI wrote it” into a run you can repeat, compare, cost, and prove.

what it gives you

Four things, one layer

01

Every frontier model

Claude Code, Codex, Kiro, and more through the same topology. A/B on identical work, or phase-split a run — design on one model, build on another. No lock-in.

02

Your own subscriptions

Subscription-funded by default — metered API is an explicit opt-in. Mix vendors and funding, with one cross-vendor cost ledger showing what each run spent.

03

A real method

Aligned with the AWS AI-DLC lifecycle — inception → construction → operations — a structured, repeatable workflow. Method-agnostic too: BMAD, or your own.

04

Proof you can ship

Deterministic gates decide “green,” not the agent. Every run is a signed, checksummed evidence pack — tamper-evident and attributable, verifiable after it leaves your machine.

Where this is

Cadora is a working reference implementation of the neutral-conductor idea — shipped, tested, backend-neutral. It is offered as the capstone of an exploration, not a finished commercial platform. The LLM-as-judge gate is a deliberate stub; the load-bearing parts — the gates, the integrity scan, the signed evidence, the cost ledger — are real. The vision stands on its own; the tool shows it holds.
